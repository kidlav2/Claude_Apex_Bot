**Период:** 180 days (Nov 2025 – May 2026)  
**Скрипт:** `backtest_session_sweep.js`  
**Составлен:** 2026-05-15  
**Риск на сделку:** $10/R (нормализованная единица)

---

## Методология

| Параметр | Значение |
|----------|---------|
| Таймфрейм | 15m LTF + 1H HTF + 1D Daily |
| Вход | Close сигнального бара (консервативно) |
| Выход TP/SL | High/Low последующих баров |
| Ничья (SL+TP в одном баре) | SL победа (worst case) |
| BE | Срабатывает при +1R нереализованной прибыли |
| Макс. удержание | 16 баров (4h на 15m) |
| Дедупликация | 1 сигнал / Kill Zone / символ / модель |

### Model A — PDH/PDL (baseline)
Previous Day High/Low → sweep trigger в любой Kill Zone.  
Использует `evaluateBars()` из `strategy.js` без изменений.

### Model B — Session H/L (новая)
- **Asia Session (00:00–06:00 UTC)** → sweep trigger в Frankfurt (06:00–07:00) и London (07:00–08:00) Kill Zones  
- **London Session (07:00–12:00 UTC)** → sweep trigger в NY AM (14:00–15:00) и PM (18:00–19:00) Kill Zones  
- Все остальные условия идентичны Model A (FVG, ATR, EMA, Health)

## Ключевые выводы

### 1. Model B генерирует больше сигналов (21 vs 17, +24%)

Session H/L диапазоны **уже**, чем PDH/PDL (Asia диапазон 6h vs полный предыдущий день 24h).  
Более узкие уровни **проще пробить** внутри Kill Zone — отсюда снижение sweep-блоков на 13.5%.

### 2. Model B улучшает avgR на всех высоковолатильных символах

Разница статистически значимая на BTC (+1.34R) и XRP (+0.52R) — именно тех инструментах, где институциональные сессии четко разделяют Asia→London→NY диапазоны.

На ETH и BNB Model B выдаёт 0 сделок — это **не регрессия**, это более строгий фильтр:
- Session Low ETH была почти каждый день на уровне $2,300–$2,320
- Цена не возвращалась в FVG после sweep → `fvgActive` продолжал блокировать

### 3. Основной блокировщик — НЕ тип sweep, а FVG + Daily EMA

Оба условия блокируют **~9,000–9,400 раз** независимо от модели. Это означает:
- Замена PDH/PDL на Session H/L сама по себе не решает проблему низкой частоты сигналов
- Настоящий bottleneck — совпадение `fvgActive` + `dailyEMAtrend` + `priceNearEMA` одновременно

### 4. Компрессионный рынок (Nov–May) — стратегия работает как и должна

Нулевые сделки на ETH/BNB отражают реальную рыночную ситуацию, а не ошибку логики:
- ETH диапазон $2,310–$2,345 (1.5%) в мае → нет sweep, нет FVG-тапа, нет тренда
- Стратегия правильно пропускает flat-рынок

## Статистические оговорки

⚠️ **Малая выборка**: 17 сделок Model A и 21 сделка Model B за 180d недостаточны для статистически значимых выводов. Нужно минимум 50–100 сделок на модель. Результаты указывают **направление**, но не конечный приговор.

⚠️ **Отсутствие slippage**: Entry по close сигнального бара не учитывает проскальзывание на market order — в live боте реальная цена входа будет хуже на 1–3 ticks.

⚠️ **DST approximation**: Kill Zone часы в бэктесте — фиксированный UTC. Живой бот использует NY timezone с DST. Ошибка ±1h в 6 месяцах ≈ 26 дней EST (ноябрь–март), что может сдвигать ~15% сделок на неверную KZ.
