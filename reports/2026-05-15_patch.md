## Task 1 — Price Floor Guard (`screener.js`)

**Проблема:** ENAUSDT @ $0.13 прошёл скринер и все 17 условий стратегии, но из-за коллапса точности форматирования (`formatStep`) SL и TP оба оказались равны цене входа ($0.13). Bracket с нулевым расстоянием самотриггерится мгновенно.

**Изменение:** `SCREENER.minPrice` default: `0.01` → **`1.00`**

```javascript
// screener.js, SCREENER object
minPrice: parseFloat(process.env.SCREENER_MIN_PRICE || "1.00"),
```

Применяется в двух местах:
- `listEligiblePerpetuals()` — первичная фильтрация по 24h тикеру
- `revalidateWatchlist()` — per-tick revalidation внутри Kill Zone

Переопределяется через `.env`: `SCREENER_MIN_PRICE=0.50`

## Task 2 — Degenerate Bracket Guard (`binance-futures.js`)

**Проблема:** Даже если токен с низкой ценой прорвётся через скринер, функция `placeOcoBracket()` размещала ордера без проверки расстояния между триггером и текущей ценой.

**Изменение:** добавлена проверка перед размещением TP и SL. Если расстояние между триггером и текущей mark-ценой меньше `MIN_BRACKET_TICKS × tickSize` — выбрасывается исключение, ни один ордер не размещается.

```javascript
// binance-futures.js — новая константа
const MIN_BRACKET_TICKS = parseInt(process.env.MIN_BRACKET_TICKS || "5", 10);

// Внутри placeOcoBracket(), до создания TP/SL params:
const tickerNow = await publicRequest("/fapi/v1/ticker/price", { symbol });
const markNow = numRequired(tickerNow.price, "markPrice");
const minDist = MIN_BRACKET_TICKS * filters.tickSize;

if (Math.abs(parseFloat(tpTrigger) - markNow) < minDist)
  throw new Error(`Degenerate bracket — TP trigger $${tpTrigger} is X ticks from mark`);
if (Math.abs(parseFloat(slTrigger) - markNow) < minDist)
  throw new Error(`Degenerate bracket — SL trigger $${slTrigger} is X ticks from mark`);
```

**Поведение при срабатывании:** исключение перехватывается в `bot.js → processSymbol()` как `ocoErr`. Активируется ghost-position guard: попытка emergency market close + Telegram-алерт оператору.

Переопределяется через `.env`: `MIN_BRACKET_TICKS=10`

## Task 3 — Sweep Age Window (`strategy.js`)

**Проблема:** `maxSweepAgeBars = 15` (3.75h на 15m) блокировал 98.5% всех setup'ов в период компрессии (May 12–14): диапазон ETHUSDT ~1.5%, PDH/PDL не пробивались ни разу за Kill Zone.

**Изменение:** `maxSweepAgeBars`: `15` → **`20`** (5h на 15m). Теперь env-tunable.

```javascript
// strategy.js, STRATEGY object
maxSweepAgeBars: parseInt(process.env.MAX_SWEEP_AGE_BARS || "20", 10),
```

**Обоснование:** 20 баров = 5h на 15m = текущая KZ + предыдущая KZ в тот же день. Sweep остаётся в пределах дневной структуры, не принимая стейл-данные прошлых суток.

Переопределяется через `.env`: `MAX_SWEEP_AGE_BARS=15` (откат) или `MAX_SWEEP_AGE_BARS=24` (6h).

## Task 4 — SIGTERM Telegram Alert (`bot.js`)

**Проблема:** GCP preemption 14 мая обрезал сессию в 14:19 UTC без уведомления. `.bot.lock` освобождается корректно через `process.on("exit")`, но session summary и сигнал оператору не отправлялись.

**Изменение:** `process.on("SIGTERM", ...)` заменён на асинхронный обработчик с жёстким дедлайном.

```javascript
process.on("SIGTERM", () => {
  const kz = activeKillZone();
  const msg = [
    `⚠️ *Bot SIGTERM — GCP Preemption Detected*`,
    `Instance shut down at \`${new Date().toISOString()}\`.`,
    kz ? `Active Kill Zone: *${kz}* — session may be incomplete.`
       : `No active Kill Zone at time of signal.`,
    `Process lock released. CSV and journal state preserved.`,
  ].join("\n");

  const deadline = setTimeout(
    () => { console.log("SIGTERM deadline exceeded"); process.exit(143); },
    parseInt(process.env.SIGTERM_ALERT_TIMEOUT_MS || "5000", 10),
  );

  sendTelegram(msg)
    .catch(() => {})
    .finally(() => { clearTimeout(deadline); process.exit(143); });
});
```

**Гарантии:**
- Если Telegram не настроен — `sendTelegram` возвращает немедленно, `.finally()` вызывает `process.exit(143)` без задержки
- Если сеть недоступна — 5-секундный дедлайн форсирует выход независимо
- `process.on("exit", releaseLock)` всегда вызывается синхронно перед завершением

Переопределяется через `.env`: `SIGTERM_ALERT_TIMEOUT_MS=3000`
