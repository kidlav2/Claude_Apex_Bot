# Claude Trading Bot — Project Rules

## КРИТИЧЕСКОЕ ПРАВИЛО: Проверка модели перед торговлей

**Две фазы работы — две разные модели:**

| Задача | Модель |
|--------|--------|
| Установка, настройка, отладка, планирование | Sonnet — ОК |
| Анализ графика, сигналы, обоснование, journal | **Только Opus 4.7** |

Если пользователь просит торговый анализ, а активна не `claude-opus-4-7` — выведи предупреждение и остановись:

> Внимание: Активна модель [НАЗВАНИЕ МОДЕЛИ]. Для торгового анализа переключись на Opus 4.7 командой `/model`. Торговые рекомендации заблокированы.

Никаких анализов графиков, сигналов на вход/выход, обоснований сделок или записей в trading_journal.md до подтверждения переключения на Opus 4.7.

---

## Архитектура

- **Данные:** Binance public API (бесплатно, без авторизации)
- **Индикаторы:** EMA(8), VWAP (сессионный), RSI(3)
- **Исполнение:** Binance API (spot)
- **TradingView MCP:** ~/tradingview-mcp-jackson — 78 инструментов для чтения чарта и управления TV Desktop через CDP (порт 9222)
- **Журнал:** trading_journal.md — обоснование + состояние индикаторов для каждого запуска
- **Tax CSV:** trades.csv

## Запуск бота

```bash
node bot.js           # один запуск
node bot.js --tax-summary   # сводка по сделкам
```

## Переменные окружения (.env)

- `PAPER_TRADING=true/false` — режим бумажной торговли
- `SYMBOL=BTCUSDT` — инструмент
- `TIMEFRAME=4H` — таймфрейм
- `PORTFOLIO_VALUE_USD=1000`
- `MAX_TRADE_SIZE_USD=100`
- `MAX_TRADES_PER_DAY=3`
